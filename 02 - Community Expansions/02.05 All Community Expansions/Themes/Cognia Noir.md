## Customization Options (Style Settings Plugin) 

**Look and feel**: 
- Accent color: Used for links, focus rings, and selection. Defaults clear WCAG AA against each theme's canvas.
- Inline code color: Color of `inline code` text. Defaults clear WCAG AA against each theme's code background.
- Background tone: Charcoal is default. Pure black is OLED-friendly. Warm charcoal adds a touch of warmth.
- Sidebar contrast: How the sidebars separate from the main editor.
- Translucency tint: Strength of the dark wash layered over macOS vibrancy. Pure shows raw vibrancy; Heavy dims it for legibility on bright wallpapers. Requires Settings -> Appearance -> Translucent window enabled (otherwise this knob has no effect).
- Heading scale
- Editor font size
- UI font: Font for UI chrome (sidebars, tabs, buttons, dialogs).
- Note font: Font for note content (editor and reading view).
- Heading font: Font for H1-H6. Pair with the note font above for a matched look.
- Note line height: Line spacing for note content. 1.65 is the default comfortable reading density.
- Line width: Maximum width of the note content area. Obsidian default is 700px.
- Bubble nav buttons: Collapses the file explorer / bookmarks action row (new note, new folder, sort, expand) into a small pill that expands on hover. Inspired by Velocity.
